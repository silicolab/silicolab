## 目标

从一份全新的 602 原子 ubiquitin 重原子条目开始，在 Console 中补加 570 个氢并保留
残基到原子映射，再使用 **Modify PTM** 向 A 链 Ser20 连接一个磷酸基团。

固定计数序列为：

```text
602 / 608 / 76 -> 1,172 / 1,178 / 76 -> 1,177 / 1,183 / 77
```

最终结果有两条链记录和一个连通分量。

## 固定样例与所需输入

下载 [`PS-UBIQUITIN-01`](../../../samples/ubiquitin.pdb)。这是从 1UBQ 清理得到的 A 链
重原子模板，不是原始数据库沉积文件或完整生物组装。水、离子、氢、其他链、`CRYST1`
和 `CONECT` 均已移除。

“全新”表示该导入条目没有运行过准备或修饰操作。导入文件后，明确使其成为
active entry（当前条目），并核对：

| 检查项 | 起始值 |
| --- | --- |
| 原子 / 键 / 残基 | `602 / 608 / 76` |
| 分子式 | `C378N105O118S` |
| 链 / 晶胞 | A 链 / `none` |

非糖链 PTM 连接要求目标重原子上存在可被置换的显式氢。因此，本工作流使用保留映射的
Console 命令 `hydrogen add`，不使用新增氢未纳入映射的 GUI 蛋白质准备结果。

## 磷酸化 A:Ser20

| 步骤 | 操作 | 可观察结果 |
| --- | --- | --- |
| 1 | 选择 **File > Open File...**（文件 > 打开文件），导入全新的 `ubiquitin.pdb`，并明确激活该条目 | Details（详细信息）显示 602 个原子、608 条键、76 个残基、分子式 `C378N105O118S` 和晶胞 `none` |
| 2 | 打开 **Console**，输入 `hydrogen add` 并运行一次 | Console 命令记录显示 `added 570 hydrogen(s)`；当前条目在原位补氢并保留残基映射。此时有 1,172 个原子、1,178 条键、76 个残基，分子式为 `C378H570N105O118S`，晶胞为 `none` |
| 3 | 保持这份精确的 1,172 原子中间态为当前条目。打开 **Launch**（启动），展开 **Structure Editing**（结构编辑），选择 **Modify PTM** | **Modify Protein (PTM)** 面板打开，并以保留映射的中间态作为宿主 |
| 4 | 把 **Modification** 设为 **Phosphorylate**，**Chain** 设为 `A`，**Residue #** 设为 `20`（Ser20）。保持 **Result name** 为空以使用自动命名 | 面板显示 `Phosphorylate`、A 链和 20 号残基；此时尚未创建结果 |
| 5 | 选择 **Apply**（应用） | 新结果创建并成为当前条目。临时状态提示显示 `Phosphorylate applied` 和随项目变化的条目编号；**Activity**（活动）将 **Modify Protein (PTM)** 记录为 `Completed` |
| 6 | 检查当前结果 | 结果有 1,177 个原子、1,183 条键、77 条残基记录、2 条链记录、1 个连通分量，分子式为 `C378H571N105O121PS`，且没有周期性晶胞 |

不要用条目编号或自动名称识别结果，两者都取决于当前项目。应根据输入状态、目标位点、组成
和计数识别结果。

## 核对结果与连接

| 状态 | 原子 | 键 | 残基 | 分子式 | 其他结构状态 |
| --- | ---: | ---: | ---: | --- | --- |
| 全新重原子输入 | 602 | 608 | 76 | `C378N105O118S` | A 链；无晶胞 |
| 保留映射的 `hydrogen add` 中间态 | 1,172 | 1,178 | 76 | `C378H570N105O118S` | 原位新增 570 个氢；无晶胞 |
| A:Ser20 磷酸化结果 | 1,177 | 1,183 | 77 | `C378H571N105O121PS` | 两条链记录；一个连通分量；无晶胞 |

77 条记录由 76 个宿主氨基酸残基和 1 个非标准 `PO4` 修饰残基组成。两条链记录分别属于
宿主链和修饰片段，不是两个蛋白质亚基。Ser20 与磷酸片段之间的 `OG-P` 共价连接把两者
合并为一个连通分量。

生成的几何结构还存在约 `1.242 A` 的宿主-片段短接触，对应映射到 `A:Glu18` 的一个
新增氢与修饰原子 `PO4 O1P`。排除由一条、两条或三条共价键分隔的宿主-片段原子对后，
该距离仍然存在。它不是力场对非键接触的定义，也不能证明结构无冲突；它表示仍有未处理的
重叠风险。

## 支持的修饰类型与锚点

这些选项用于结构连接，不是生物学位点预测。

| 修饰 | 可用残基与锚点 | 选择边界 |
| --- | --- | --- |
| **Phosphorylate** | Ser `OG`、Thr `OG1`、Tyr `OH` 或 His `NE2` | 面板对 His 使用 `NE2`；本工作流使用 Ser20 `OG` |
| **Acetylate** | Lys `NZ`，或所选链第一个残基的骨架 `N` | 启用 N 端选项时，所选残基必须是该链首残基 |
| **Methylate** | Lys `NZ` 或 Arg `NH1` | 可选 mono、di 和 tri；残基必须与锚点匹配 |
| **Lipidate** | palmitoyl、farnesyl 或 geranylgeranyl 使用 Cys `SG`；myristoyl 使用链首 Gly 的骨架 `N` | myristoyl 目标必须是 Gly，且是所选链的首残基 |
| **Ubiquitinate** | 宿主 Lys `NZ` | 可选择 Ubiquitin、SUMO 或 NEDD8；宿主锚点仍为 Lys `NZ` |
| **Glycosylation** | N-linked 使用 Asn `ND2`；O-linked 使用 Ser `OG` 或 Thr `OG1` | 锚点残基决定连接类型 |

**Modify PTM** 只接受链标识符和整数残基号，不能选择带 insertion code 的残基 ID。
不要丢弃 insertion code，再用同号普通残基替代原目标。

## 恢复到已知状态

| 现象 | 恢复方法 |
| --- | --- |
| 导入计数不是 `602 / 608 / 76`，或不能确定当前条目 | 重新导入样例，明确激活新条目，并同时核对计数和 `C378N105O118S` |
| `hydrogen add` 在错误条目上运行，或没有得到 `1,172 / 1,178 / 76` | 停止使用该条目；重新导入并激活全新源条目，核对起始状态，再只运行一次命令 |
| Apply 显示 `modification failed`、找不到残基，或找不到可置换的锚点氢 | 重新构建精确的保留映射中间态，激活它，并再次核对 A 链和 20 号残基 |
| 已经用错误类型、链或残基创建结果 | 不在该结果上继续；重新激活已核对的 1,172 原子中间态，再按固定设置重做 |
| 目标残基带 insertion code | 停止这条 GUI 路线，并保持中间态不变 |

## 科学限制

生成的 PTM 几何结构是理想化且未最小化的。`Completed`、分子式、计数和已存储的 `OG-P`
键只说明 SilicoLab 完成了结构连接；它们不能指定生物修饰状态、酶特异性、占有率、最终电荷
或完整力场拓扑。

在参数化或模拟前，必须核对预期生物修饰、化学连接、立体冲突及其他重叠、质子化和电荷。
建立并审核完整的连接拓扑和力场参数，然后最小化或以其他方式松弛整个结构。生成条目不能
直接作为 MD 输入。
