## 目标

从 602 原子的 ubiquitin 重原子模板创建一个独立的补氢条目。核对
`602 / 608 / 76` 到 `1,172 / 1,178` 的变化，确认新增 570 个氢，并了解 GUI 结果的
残基到原子映射限制。

只有后续操作需要新增氢保留残基归属时，才从一份全新的源条目运行 Console 命令
`hydrogen add`。

## 固定样例

下载 [`PS-UBIQUITIN-01`](../../../samples/ubiquitin.pdb)。这是从 1UBQ 清理得到的 A 链
重原子模板，不是原始数据库沉积文件或完整生物组装。水、离子、氢、其他链、`CRYST1`
和 `CONECT` 均已移除。

导入文件后，双击其条目，使其成为 active entry（当前条目）。继续前同时核对以下值：

| 检查项 | 起始值 |
| --- | --- |
| 原子 / 键 / 残基 | `602 / 608 / 76` |
| 分子式 | `C378N105O118S` |
| 链 / 晶胞 | A 链 / `none` |

## 在 GUI 中准备蛋白质

打开 **Launch**（启动），展开 **Molecular Dynamics**（分子动力学），选择
**Prep Protein**（蛋白质准备）。保持 **Add missing hydrogens**（补加缺失氢原子）启用，
然后选择 **Prepare**（准备）。

| 步骤 | 操作 | 可观察结果 |
| --- | --- | --- |
| 1 | 选择 **File > Open File...**（文件 > 打开文件），导入 `ubiquitin.pdb`，并明确激活新导入的条目 | Details（详细信息）显示 602 个原子、608 条键和分子式 `C378N105O118S`；Sequence（序列）包含 76 个残基；晶胞为 `none` |
| 2 | 在 **Launch > Molecular Dynamics** 中选择 **Prep Protein** | **Protein Preparation** 面板打开，**Add missing hydrogens** 已启用；此时尚未创建结果 |
| 3 | 选择 **Prepare** | 临时状态提示显示 `Protein prepared: added 570 hydrogen(s) (new entry)`；**Activity**（活动）将 **Protein Preparation** 记录为 `Completed`；新结果创建并成为当前条目 |
| 4 | 在 Details 中检查新结果，并确认源条目仍独立存在 | 结果有 1,172 个原子、1,178 条键，分子式为 `C378H570N105O118S`，晶胞为 `none`；源条目没有被原位替换 |

`Completed` 只表示软件操作已经结束，不表示该结构已经通过科学验证。

## 对比输入与结果

蛋白质元数据包含残基到原子映射，当前实现称为 `biopolymer mapping`。它记录每个已映射
原子属于哪个残基。

| 状态 | 已存储检查项 | 残基到原子映射 |
| --- | --- | --- |
| 重原子输入 | `602 个原子 / 608 条键 / 76 个残基`；`C378N105O118S`；晶胞 `none` | 覆盖 76 个残基中的全部 602 个输入原子 |
| GUI 结果 | `1,172 个原子（+570） / 1,178 条键（+570）`；`C378H570N105O118S`；晶胞 `none` | 复制原有残基记录和前 602 个原子的映射，但没有把映射扩展到新增的 570 个氢 |

每个新增氢形成一条新键，因此原子数和键数都增加 570。这只核对组成和连接增量，不检查
立体重叠，也不进行几何松弛。

**Prep Protein** 面板目前执行通用的价态补氢，并把结果存为新条目。只需要独立的补氢
结构，而且新增氢缺少映射不会影响后续工作时，可以使用该结果。

## 使用 Console 保留残基映射

如果后续结构操作需要新增氢的残基归属，请从新导入的 `PS-UBIQUITIN-01` 重新开始。
不要在 1,172 原子的 GUI 结果上运行该命令。

| 步骤 | 操作 | 可观察结果 |
| --- | --- | --- |
| 1 | 重新导入样例并明确激活这份全新源条目 | Details、Sequence 和晶胞恢复为 `602 / 608 / 76`、`C378N105O118S` 和 `none` |
| 2 | 打开 **Console**，输入 `hydrogen add` 并运行 | Console 命令记录显示 `added 570 hydrogen(s)`；当前条目被原位修改，而不是再创建一个准备结果 |
| 3 | 再次检查 Details 和 Sequence | 当前条目为 `1,172 / 1,178 / 76`，分子式为 `C378H570N105O118S`，晶胞为 `none`；每个新增氢映射到其所连接重原子的残基 |

该命令保留下一页 PTM 路线等结构操作所需的映射，但不会补充通用补氢所省略的化学决策。

## 恢复到已知状态

如果准备或修改了错误条目，请停止使用该结果。重新导入样例，明确激活新导入条目，并且
只在 `602 / 608 / 76`、`C378N105O118S` 和晶胞 `none` 同时出现后继续。

| 现象 | 恢复方法 |
| --- | --- |
| Protein Preparation 面板中的输入或选项错误 | 选择 **Cancel**（取消），重新导入并激活样例，再次打开 **Prep Protein** |
| GUI 从错误源条目创建了结果 | 不再使用该结果，从新导入重新开始；源条目没有被覆盖 |
| `hydrogen add` 在错误条目上运行 | 停止使用该条目；重新导入样例，不要假定 Undo 已经恢复固定起点 |
| Console 显示 `hydrogen add requires an open entry`，或计数不同 | 激活一份新导入，并在运行命令前重新核对全部起始值 |

## 科学限制

两条路径都只执行通用的价态补氢。它们不会选择质子化状态、组氨酸互变异构体、末端状态、
二硫键或电荷，也不会补加缺失重原子；同时也不会进行蛋白质专用的冲突检查或几何最小化。

在参数化或模拟前，应检查缺失重原子、残基化学、质子化、组氨酸状态、末端、二硫键、电荷、
立体接触和力场拓扑，并按需要最小化或以其他方式松弛几何结构。原子数、键数、完成状态和
保留的残基映射都不能使结构直接成为可用于 MD 的输入。
