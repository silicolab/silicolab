## 目标

在空项目中构建 `4 x 4 x 1` graphene，再用明确的 `2 x 2 x 1` 重复因子原位扩展当前条目。
预期变化是从 `C32`、32 个原子和 48 条已存储键变为 `C128`、128 个原子和 192 条已存储键。

## 构建起始晶胞

打开 **Launch**（启动），展开 **Structure Builder**（结构构建），选择 **Nanosheet Builder**
（纳米片构建器）。明确输入每个值：

| 设置 | 数值 |
| --- | --- |
| Sheet | `Graphene` |
| Sublattice A / B | `C` / `C` |
| Lattice a | `2.46 Å` |
| Buckling | `0 Å` |
| Interlayer spacing (A) | `12 Å` |
| Supercell | `4 x 4 x 1` |

对于 zero-buckling graphene，**Interlayer spacing (A)** 就是沿 `c` 方向的周期镜像间隔，
因此会得到 `12.000 Å` 的已存储 `c` 长度。

选择 **Build**（构建）。界面显示临时成功提示，**Activity**（活动）把
**Nanosheet Builder** 记录为 `Completed`，新条目成为当前条目。只有 Details（详细信息）
同时显示以下检查项时才继续：

| 检查项 | 预期值 |
| --- | --- |
| 分子式 / 原子 / 已存储键 | `C32` / 32 / 48 |
| 已存储连接图 | 1 个连通分量 |
| 晶胞长度 | `9.840 x 9.840 x 12.000 Å` |
| 晶胞角 | `90.000 / 90.000 / 60.000°` |

临时提示和 `Completed` 状态只表示构建操作已经结束。已存储条目及其 Details 数值才是
持久的结果检查项。

## 扩展当前条目

保持已核对的 `C32` 条目为当前条目。在 **Launch** 中展开 **Structure Editing**（结构编辑），
选择 **Expand Supercell**（扩展超胞），把三个重复因子设为 `2`、`2` 和 `1`。不要依赖
面板中已有的数值。

选择 **Expand**（扩展）。该操作原位修改当前条目。临时提示显示
`Expanded to 2x2x1 supercell (128 atoms, 192 bonds)`，**Activity** 把
**Supercell Expansion** 记录为 `Completed`。

| 检查项 | 扩展前 | 扩展后 |
| --- | ---: | ---: |
| 分子式 | `C32` | `C128` |
| 原子 | 32 | 128 |
| 已存储键 | 48 | 192 |
| 已存储连接图 | 1 个连通分量 | 1 个连通分量 |
| 晶胞长度 | `9.840 x 9.840 x 12.000 Å` | `19.680 x 19.680 x 12.000 Å` |
| 晶胞角 | `90.000 / 90.000 / 60.000°` | `90.000 / 90.000 / 60.000°` |

重复数乘积为 `2 x 2 x 1 = 4`，因此原子数和已存储键数都变为原来的 4 倍。
`a`、`b` 长度加倍，`c` 和所有晶胞角不变。

## 晶胞、坐标与显示

SilicoLab 分别存储晶胞和原子位置。晶胞长度使用 Å，晶胞角使用度。Cartesian（笛卡尔）
坐标以 Å 表示位置；fractional（分数）坐标以晶格矢量为基底，因此 `(1, 0, 0)` 表示一个
完整的 `a` 矢量。

对周期性结构，**Edit Structure...**（编辑结构）最初显示分数坐标，也可切换为 Cartesian。
切换表示方式不会移动原子。分数坐标也不必位于 `[0, 1)`；只有 wrap 操作会把每个分量
映射到该区间。

Details 中的六个数值是已存储晶胞。**Style > Scene > Unit cell**（样式 > 场景 > 晶胞）
只控制视口中晶胞线框的显示与隐藏。隐藏线框不会删除晶胞，显示线框也不能证明已存储数值正确。

## Wrap 不等于扩展

| 操作 | 结果 |
| --- | --- |
| **Edit > Edit Structure... > Wrap into cell > Apply** | 在编辑草稿中 wrap 原子，并在应用编辑时根据 wrap 后的坐标重新计算键 |
| **Launch > Structure Editing > Wrap Into Cell** | 原位 wrap 当前周期性条目，同时保留其已存储键 |
| **Launch > Structure Editing > Expand Supercell** | 复制原子和已存储连接图，按重复因子放大晶胞，并原位修改当前条目 |

两条 wrap 路径都逐个处理原子，而不是把每个完整分子作为整体平移。因此，跨越周期边界的
分子可能显示为分处第一晶胞的两侧。Wrap 不增加原子数或晶胞长度；扩展会复制结构并放大晶胞。

## 恢复到已知状态

| 现象 | 恢复方法 | 继续前的检查 |
| --- | --- | --- |
| Graphene 设置或起始计数错误 | 取消面板，在空项目中按上方全部固定值重新构建 | `C32`、32 个原子、48 条键、1 个分量和原始晶胞一致 |
| 使用了错误重复因子 | 如果扩展是最后一次编辑，立即 Undo；否则重新构建起始结构 | 当前条目回到完整的 `C32` 检查点 |
| 扩展报告结构没有周期性晶胞 | 停止，激活已核对的 graphene 条目，并检查 Details | 6 个晶胞参数全部存在；只有可见线框还不够 |
| Wrap 拆开了分子或意外改变键 | 立即 Undo；无法确定先前状态时重建固定起点 | 分子式、原子、键、分量和晶胞都与操作前一致 |
| 只是不见线框 | 在 Style 中重新启用 **Unit cell** | Details 保持不变 |

只要分子式、计数、连通分量或任何晶胞参数与对应检查点不同，就停止工作流。不要从操作前
状态不确定的结构继续。

## 科学限制

构建和扩展不会弛豫或优化几何结构。扩展只复制已记录的原子、晶胞和连接图；不会推断缺失的
周期镜像键、重建晶体学对称性、解析部分占位、规范化源结构或检查全部重叠。

晶胞线框、复制算术、分子式和 `Completed` 状态都只是结构与软件检查项。它们不能证明能量
稳定性、缺陷模型有效、可直接用于模拟或具有实验意义。
