## 建立可复现样例

下载[ubiquitin 样例](../../../samples/ubiquitin.pdb)。它是经过清理的 1UBQ A 链重原子模板，
含 602 个原子和 76 个残基。水、离子、氢、其他链、`CRYST1` 和 `CONECT` 已被移除，
因此 SilicoLab 会在无周期边界的条件下推断化学键。该样例不是原始沉积文件，也不是完整
生物组装。

1. 选择 **File > Open File...**（文件 > 打开文件）并打开样例。
2. 确认 Entries 中出现分组 `UBIQUITIN (1UBQ chain A, heavy atoms)` 和条目
   `UBIQUITIN`，且该条目成为 active entry（当前条目）并显示在中央视口。
3. 打开底部 **Sequence**（序列）页签，确认显示 chain A 和 76 个残基。
4. 单击序列位置 48 的残基 `K`，确认 Lys48、视口中的 9 个重原子以及 Sequence 计数
   同步高亮。

这种同步显示只检查选择行为，不证明样例具有科学完整性或适合计算。

## 区分三个选择范围

| 范围 | 含义 | 可见检查 |
| --- | --- | --- |
| 当前条目 | 中央视口显示、普通单结构任务使用的唯一条目 | 对应标签页处于当前状态，结构显示在视口中 |
| 侧栏已选条目或分组 | Export 和侧栏批量动作使用的 Entries 高亮范围 | 条目行或分组标题高亮，但不会自动改变当前条目 |
| 已选原子或残基 | 当前条目内的原子索引；选择残基会展开为该残基的原子 | 视口中的原子高亮；兼容的 Sequence 行显示相同残基高亮和计数 |

## 选择或激活条目

| 操作 | 结果 |
| --- | --- |
| 单击条目 | 选择侧栏条目行，但不让它成为当前条目 |
| 双击条目 | 选择该条目，打开或激活对应标签，并让它成为当前条目 |
| `Cmd`/`Ctrl`-单击 | 在侧栏选择中切换条目或分组，不改变视口 |
| `Shift`-单击 | 选择从当前锚点到目标的可见范围 |

折叠分组的成员以及被搜索隐藏的行不属于 Shift 范围。应把实际高亮的行作为权威侧栏范围。

## 在视口中选择原子

| 操作 | 原子选择结果 |
| --- | --- |
| 单击一个原子 | 用该原子替换当前原子选择 |
| `Cmd`/`Ctrl`-单击原子 | 在当前选择中切换该原子 |
| 单击视口空白区域 | 原子选择保持不变 |
| **Selection > Clear Selection** 或 `Cmd/Ctrl+Shift+A` | 清除视口和 Sequence 高亮，但不清除 Entries 侧栏选择 |

临时状态反馈会报告已选原子数或确认已经清除原子选择。原子选择只属于当前条目，不是
Export 范围。

## 在 Sequence 中选择残基

| 操作 | 结果 |
| --- | --- |
| 单击一个残基 | 用该残基中的全部有效原子替换原子选择 |
| `Cmd`/`Ctrl`-单击残基 | 切换该残基中的全部原子 |
| 在同一条链中 `Shift`-单击第二个残基 | 选择从锚点到目标的链内残基范围 |
| 拖过多个残基单元格 | 选择拖动范围触及的残基；修饰键可以添加、切换或移除 |
| 双击链标识区 | 用该链当前显示的全部残基替换选择 |

Sequence 是当前原子选择的另一个视图，不是独立保存的选择列表。其标题显示已选残基和原子
数量，视口则高亮同一组原子。

只有当前条目含有与现有原子数兼容的蛋白质或核酸元数据时，Sequence 才显示残基行。
否则会显示 `No protein or nucleic-acid sequence metadata for the active structure.`。
这条消息本身不能证明文件导入失败；应先检查当前条目、结构类型和原子数。

## 导入或切换条目后重新检查范围

文件导入成功时会清除原子选择。切换当前条目则不保证选择为空：在新结构中仍然有效的索引
可能保留，超出原子数的索引会被移除。

切换条目后应同时检查视口和 Sequence。后续操作若依赖精确范围，先使用
**Clear Selection**，再重新选择目标原子或残基。
