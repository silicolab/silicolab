## 理解工作区边界

SilicoLab 启动后进入 **Scratch temporary workspace**（Scratch 临时工作区）。无需先选择
项目目录，就可以在 Scratch 中查看、导入或准备结构。但其中的内容是临时的：如果没有创建
项目，关闭 SilicoLab 后不会保留 Scratch 工作区。

| 工作区 | 如何辨认 | 持久化边界 |
| --- | --- | --- |
| Scratch | 欢迎区标题显示 `Scratch temporary workspace` | 没有项目数据库；创建项目时，当前已经提交的 Scratch 条目和任务状态会带入新项目 |
| Project（项目） | 标题栏显示项目名 | 条目、项目任务记录和已保存的应用状态存放在所选项目根目录下 |

SilicoLab 项目是一个目录，而不是单个项目文件。项目根目录还可能引用目录外的运行数据，
因此它不会自动成为自包含归档。不要手工编辑项目目录中的数据库或其他由 SilicoLab 管理的
文件。

## 创建项目时不要替换已有目录

1. 选择 **File > Create a new project...**（文件 > 新建项目）。
2. 选择可写的父目录，并输入唯一的项目名。
3. 确认前，检查最终目标目录**尚不存在**。
4. 确认创建，然后检查标题栏是否显示新项目名。

项目选择器可能看起来像文件保存对话框，但 SilicoLab 会把所选名称用作新目录。如果同名
目录已经存在，或系统显示替换、覆盖提示，请选择 **Cancel**（取消）。改用唯一名称或空的
父目录。不要通过“创建项目”来接管、合并或替换已有目录。

从 Scratch 创建项目不一定得到空项目。已经提交的 Scratch 条目和任务状态会带入新项目。
仍然打开的编辑器和未提交表单各自有 Apply、Build、Run 或 Cancel 边界，应先明确处理。

## 打开已有项目

| 目的 | 操作 | 操作后的检查 |
| --- | --- | --- |
| 打开项目 | 选择 **File > Open Project...**（文件 > 打开项目），并选择项目根目录本身 | 标题栏显示项目名；已保存的 active entry（当前条目）出现在中央视口，空项目则显示空视口 |
| 重开最近项目 | 从 **File > Recent Projects**（文件 > 最近项目）选择项目名 | 标题栏切换到该项目，并且预期条目可用 |

应选择已有项目根目录，而不是它的父目录、内部子目录或结构文件。Recent Projects 中的项
只是记住的路径，不保证该目录仍然存在或仍然有效。

如果打开时报告 `not a valid SilicoLab project`，请取消操作并找到真正的项目根目录。
如果项目已经移动，直接打开当前位置的根目录。不要在旧路径新建同名目录来掩盖失效的最近
项目记录。

如果临时状态显示 `recovered: previous session did not close cleanly`，应先检查
Entries（条目）、当前结构、Activity（活动）和重要结果，再继续编辑。该消息只表示上次
没有正常关闭；它不证明数据已经修复，也不证明每个外部任务和路径都完整。发现异常时，
先保留整个项目根目录的副本，再尝试恢复。

## 把 Save Project 当作明确检查点

SilicoLab 会在后台写入一部分已经提交的更改，但并非每个打开的界面草稿都属于已提交状态。
界面中也没有一个单独指示器可以证明所有相关内容都已安全保存。

完成重要编辑后，以及切换项目或退出前，选择 **File > Save Project**（文件 > 保存项目）。
显式保存成功时会出现临时状态 `Saved project <name>`；新的反馈可能很快替换它。仅仅等待、
没有看到警告或查看 Console（控制台），都不能证明保存成功。

如果状态报告 `Project save failed:`，或离开确认显示 `Could not save project`，请留在
当前项目。检查写入权限和剩余存储空间，然后再次保存。系统级保存失败可在 **Output**
（输出）中选择 **System** 来源后复查；成功保存项目不会在那里形成持久回执。

## 了解反馈出现在哪里

- 状态区显示 `Opened project`、`Saved project`、`Closed project` 和构建成功等临时确认。
- **Activity** 是查看当前和历史任务的可见监视页签。
- **Console** 只显示通过 `.sls` 命令界面输入的命令转录，不是一般操作日志。
- **Output** 显示非命令日志。选择 **System** 来源可以复查已记录的应用、项目、存储或
  文件失败。

把最终项目名、条目、任务状态和文件作为持久检查依据，不要期待临时成功消息一直保留。

## 保存或离开前处理草稿

**警告：** 实时预览不等于已经提交的编辑。保存、打开另一项目、关闭项目或退出前，应先
明确处理每个仍然打开的编辑器或表单。

- 在 **Structure Editor**（结构编辑器）中，选择 **Apply**（应用）提交整个编辑会话，
  或选择 **Cancel** 恢复打开编辑器之前的结构。确认编辑器关闭，且视口符合所选结果。
- 在任务面板中，使用该面板自己的 **Build**、**Start**、**Run** 或 **Cancel** 控件。
  构建成功通常只显示临时状态；应检查生成的条目、任务面板、结果位置或 **Activity**，
  不要期待 Console 或 Output 保存这条成功消息。
- 如果 Sketch（绘制）或 Export（导出）等其他对话框不应继续，也应关闭或取消。

离开确认不能替代这些决定。Scratch 中有工作时，会提供 **Create Project**（创建项目）、
**Discard**（丢弃）和 **Cancel**。持久化项目中则可能提供 **Save and Open**、
**Save and Close** 或 **Save and Quit**，并同时提供 **Don't Save** 和 **Cancel**。
如果保存失败，SilicoLab 会让离开操作保持未完成；修复失败后再重试。

## 安全关闭和重开

选择 **File > Close Project**（文件 > 关闭项目）会返回 Scratch，欢迎区重新显示
`Scratch temporary workspace`。临时状态 `Closed project; opened Scratch` 只确认当时
完成了切换；这不是删除操作，也不是持久日志记录。

重开已保存项目时，可以恢复打开的标签页、各条目的视图和当前条目。侧栏多选和分组展开
状态不能作为可靠的跨会话范围。执行任何批量动作前，应重新检查高亮行、可见分组、当前
标签和中央视口。

## 谨慎备份或移动项目

任务记录属于启动它们的工作区，但所需运行数据不一定全部位于项目根目录。本地或远程运行
目录、已分离任务记录、轨迹、对接构象和其他产物可能各自带有路径或主机要求。

**警告：** 本地或远程工作仍在运行时，不要移动或重命名项目。项目中保存的标识可以在目录
移动后保持不变，但这不保证每个任务关联或已保存运行路径都会随之迁移。SilicoLab 不承诺
自动重写全部外部路径。

备份或计划移动项目时：

1. 处理所有打开的草稿并执行 **Save Project**。在保存提示仍可见时确认成功。
2. 在 **Activity** 中确认相关本地或远程任务都不处于运行或取消中状态。
3. 保持原位置可用。复制完整项目根目录，以及每个必需的外部运行目录或产物。
4. 打开复制后的根目录，并检查 Entries、当前结构、Activity、Output 和预期结果位置。
5. 在重要工作流都已重新打开并核对前，继续保留原位置和外部路径。仅仅成功打开副本，
   不能证明所有已保存引用都不再依赖旧路径。
