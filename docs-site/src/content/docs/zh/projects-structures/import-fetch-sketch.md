## 选择输入路径

| 路径 | 使用场景 | 完成检查 |
| --- | --- | --- |
| 文件导入 | 本地已有受支持的结构文件 | Entries 中出现新条目；带标识符的 PDB 还会创建分组 |
| PDB ID 获取 | 已知经典四字符 PDB ID，并且可以联网 | PDB 缓存到当前项目的 `structures/` 目录，然后导入 |
| 二维草图 | 从 SMILES 或手工绘制结构开始 | 只有选择 **Build (Save as New)** 后，才创建新的 active entry（当前条目） |

如果导入源、缓存和生成条目需要跨会话保留，请在持久化项目中操作。

## 导入本地文件

选择 **File > Open File...**（文件 > 打开文件），并选择一个受支持的结构文件。
SilicoLab 可以读取 XYZ、CIF、MOL2、SLF、GRO、PDB 和 PDBQT。导入成功后会添加条目，
让第一个导入结构成为当前条目，清除之前的原子选择，并显示临时状态反馈。

### 用双模型夹具检查 PDB 分组

下载[合成的截短双模型夹具](../../../samples/6a5j-two-model-ui-fixture.pdb)。每个模型只包含
一个 GLY 残基的 N 和 CA 原子。该文件不是科学用途的 6A5J 模型，不可用于结构解释或计算。

1. 使用 **File > Open File...** 打开该夹具。
2. 在 Entries 中展开 `SOLUTION NMR STRUCTURE OF SMALL PEPTIDE`。
3. 确认其中包含 `6A5J (model 1)` 和 `6A5J (model 2)`。
4. 确认中央视口中的当前条目是 model 1，而不是把分组高亮误当作当前条目。

带数据库标识符的 PDB 使用沉积标题作为分组名。单模型文件也会得到包含一个标识符命名条目
的分组；多模型文件则每个模型一个条目，但只有第一个模型成为当前条目。标题元数据缺失时，
SilicoLab 回退到标识符；无 HEADER 的 PDB 可以把大写文件名主干作为后备标识符。

没有数据库标识符的文件，包括普通 XYZ 和 MOL2，会作为未分组条目导入；条目名来自解析的
标题或文件信息。

> **当前限制：** 解析器可以识别多个 XYZ 或 MOL2 记录，但图形界面对没有数据库标识符的
> 文件只保留第一条记录，也不会警告后续记录被省略。如果每条记录都需要导入，应先拆成
> 独立文件。

### 通过拖放导入多个文件

原生 Open File 选择器一次接受一个文件；拖放可以同时提交多个文件。

| 结果 | 工作区中保留什么 | 恢复方法 |
| --- | --- | --- |
| 全部成功 | 每个导入文档都保留；临时状态报告打开数量 | 继续前检查新的当前条目和 Entries |
| 部分失败 | 成功导入项仍然保留；临时状态同时报告成功数和失败数 | 只修正并重新提交失败文件，避免重复条目 |
| 全部失败 | 不创建新条目；状态显示第一个打开错误 | 先修正第一个报告的格式、扩展名或权限问题，再重试 |

## 按 PDB ID 获取结构

1. 选择 **File > Fetch from PDB ID...**（文件 > 按 PDB ID 获取）。
2. 输入经典 PDB ID，例如 `1ubq`，然后选择 **Fetch**。
3. 成功后，确认对话框关闭，导入结构出现在 Entries 中。

SilicoLab 会去除 ID 两端空白、校验格式、转换为大写，并在当前项目下保存为
`structures/1UBQ.pdb`。经典 PDB ID 必须恰好包含四个 ASCII 字母或数字，并以数字开头。
例如，输入 `abcd` 后对话框会继续打开，并显示以 `Fetch failed:` 开头的临时错误。

| 情况 | 行为与恢复 |
| --- | --- |
| ID 无效、网络错误或写入失败 | 对话框和输入内容继续保留。修正 ID、网络或项目写入权限后重试 |
| 大写 ID 已有缓存 | SilicoLab 复用当前项目中的缓存文件，然后按普通文件导入 |
| 下载成功但解析失败 | 获取对话框已经关闭，并显示普通文件打开失败反馈。修复缓存文件，或用正确 ID 重试 |

缓存属于具体项目。另一个项目的 `structures/` 中存在文件，不表示当前项目已经获取成功。

## 从二维草图构建结构

1. 选择 **File > Sketch Molecule...**（文件 > 绘制分子）。草图为空时，
   **Build (Save as New)** 不可用。
2. 输入 SMILES 并选择 **Import**（导入）。成功时画布会被替换，绘制窗口显示
   `Imported SMILES (<N> atoms)`；此时还没有创建项目条目。
3. 检查或编辑图形，填写 **Title**，然后选择 **Build (Save as New)**。
4. 确认绘制窗口关闭，新条目成为当前条目，并在中央视口显示生成的三维坐标。

SMILES 解析失败时，绘制窗口显示 `SMILES error:` 并保留之前的图形。修正文本后重新导入；
不要把保留的旧图形误当作失败输入。选择 **Cancel** 可以关闭绘制窗口而不创建条目。

Build 会补入推断的氢原子，生成初始三维坐标，并且只对受支持元素尝试 UFF 松弛。含有
不受支持元素的结构仍然可以创建，但其坐标不会经过该松弛。

**科学边界：** 导入 SMILES、绘制化学键或生成三维坐标，不等于化学修复、构象搜索、
科学验证，也不证明结构已经适合研究计算。继续前应独立检查组成、形式电荷、连接关系、
立体化学和几何。
