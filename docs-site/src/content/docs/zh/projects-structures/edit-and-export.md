## 编辑当前条目

双击目标条目，并确认它成为当前条目且显示在中央视口。然后选择
**Edit > Edit Structure...**（编辑 > 编辑结构），打开 **Edit Structure**。

Structure Editor 可以：

- 修改已有周期晶胞的参数，同时保留分数坐标并更新笛卡尔坐标；
- 在 **Fractional**（分数）和 **Cartesian**（笛卡尔）坐标之间切换，并使用
  **Wrap into cell**（移入晶胞）；
- 添加或删除原子，编辑元素、电荷和坐标；
- 重新计算、添加或删除化学键，并选择 Single、Double、Triple 或 Aromatic 键类型。

它不能创建或移除晶胞、编辑链或残基元数据、优化几何或证明化学合理性。只有结构原本带有
周期晶胞时才显示晶胞控件。

**Recompute Bonds**（重新计算键）使用距离启发式规则。**Wrap into cell** 移动坐标后也会
重新计算键。两种情况下都应在应用前核对连接关系和键类型；结果不是经过验证的价态或键级。

> **手动加键注意：** **Atom A** 和 **Atom B** 使用从 0 开始的索引，而原子表从 1 开始。
> 不要把表格行号直接填入这两个字段。添加键前，应通过界面中的 Preview 核对目标元素和
> 原子对。

## 明确结束编辑会话

编辑器中的每项变化都会在实时视口中预览。应把预览视为草稿，而不是已经提交或经过科学
验证的结构。Structure Editor 内部的 **Undo** 和 **Redo** 只浏览当前编辑草稿。

| 结束方式 | 结果 | 检查 |
| --- | --- | --- |
| **Apply** | 把整个编辑会话提交为一个项目历史步骤，并关闭编辑器 | 临时状态显示 `Applied structure edits`；编辑后的结构继续显示 |
| **Cancel** | 恢复打开编辑器之前的结构，并关闭编辑器 | 临时状态显示 `Edit canceled`；原结构返回 |

Apply 后，主界面的 **Undo** 会一次撤销整个编辑会话，**Redo** 会重新应用。保存或离开项目
前，必须选择 Apply 或 Cancel，并确认编辑器关闭。不要依赖 Save Project 或离开确认替你
决定如何处理仍打开的预览。

Apply 后若要建立持久化检查点，选择 **File > Save Project**，并在临时
`Saved project <name>` 状态仍可见时确认成功。仅仅等待或查看 Console 不等于显式保存成功。

## 区分项目保存与结构导出

| 动作 | 写入位置 | 它不会做什么 |
| --- | --- | --- |
| **Save Project** | 项目根目录下的 SilicoLab 项目数据库和状态 | 不会创建独立交换文件 |
| **Export...**（导出） | 用户选择的外部文件或文件夹 | 不替代项目保存，也不会只导出视口中高亮的原子 |

重要编辑可能同时需要两个决定：先保存项目检查点，再在其他程序或工作流需要时创建外部文件。

## 在 Export 前设置条目范围

Export 读取条目，从不读取原子选择。**Selected (N)** 包含 Entries 中已选的条目行，以及
每个已选分组的全部成员。打开对话框时会捕获这份选择。

1. 在 Entries 中高亮目标条目或分组。
2. 选择 **File > Export...**。
3. 在 **Export Structures** 中选择并核对 **Selected (N)**、
   **Active structure (N)** 或 **All in project (N)**。
4. 选择格式和布局前，检查预览名称。

从条目右键菜单打开 Export 时，如果该条目已经属于侧栏选择，会保留完整选择；如果该条目
尚未选中，则把捕获范围缩小到该条目。无论入口如何，都以 Export Structures 中的范围数量
和名称为准。

## 选择格式和文件布局

可写格式为 **XYZ**、**CIF**、**MOL2**、**PDB** 和 **PDBQT**。

- 单个条目写入一个文件。
- 多个条目只有使用 XYZ 或 MOL2 时才能选择 **Combine into one file**（合并到一个文件）。
- 多个条目使用 CIF、PDB 或 PDBQT 时采用 **One file per structure**（每个结构一个文件）。
  禁用的合并选项会说明所选格式只能每个文件保存一个结构的原因。
- 多个 PDB 结构仍然可以成功导出为独立文件；被拒绝的只是把它们合并成一个 PDB。

使用 One file per structure 时，先选择目标文件夹。SilicoLab 会规划唯一且经过清理的文件名。
如果计划写入的文件已经存在，会先显示 **Overwrite existing files?** 确认。单文件或合并文件
使用系统原生保存选择器，由该选择器处理已有文件名。

导出完成后，临时状态会报告结构数和目标位置。分文件导出中若只有部分文件写入失败，还会
报告失败数和首个错误。应直接检查目标文件，不要期待 Console 或 Output 保留成功消息。

## 预期不同格式之间会损失信息

格式转换可能省略或重新解释化学键、电荷、周期晶胞、链和残基元数据、标识符或其他源信息。
文件成功写出不能证明转换无损。应在接收程序中重新打开结果，并检查原子数、元素、坐标、
连接关系、晶胞、电荷以及后续工作流依赖的元数据。

> **当前图形界面往返限制：** 合并 XYZ 或 MOL2 文件可以包含每个已导出的结构，但
> **File > Open File...** 当前只导入第一条没有数据库标识符的记录。如果需要在 SilicoLab
> 中逐个重开并核对结构，应选择 **One file per structure**，再分别打开文件。
