## 认识工作台

标题栏显示当前项目上下文。应用菜单在 Windows 和 Linux 中位于标题栏，在 macOS
中位于系统菜单栏。主侧栏在 **Entries**（条目）、
**Launch**（启动任务）和 **Style**（样式）之间切换；中央视口只显示 active entry
（当前条目）。右侧和底部是可移动的停靠区，因此已保存的布局可能不同。

| 位置 | 用途 | 操作后到哪里查看 |
| --- | --- | --- |
| 标题栏 | 确认项目名称；Windows 和 Linux 可在此打开应用菜单，macOS 则使用系统菜单栏。 | 它确认项目上下文，但不表示任务或计算已经完成。 |
| 主侧栏 | 在 **Entries** 中管理结构条目，在 **Launch** 中发起任务，在 **Style** 中调整当前结构的显示。 | 新结构通常出现在 Entries；任务本身及其状态不等同于当前条目。 |
| 中央视口 | 显示当前条目，并显示该结构中的原子选择。 | 只有成为当前条目的结构会在这里显示。 |
| 底部停靠区 | **Console**（控制台）只包含命令记录并提供 `sls>` 输入；**Sequence**（序列）显示当前结构的序列信息；**Activity**（活动）显示任务状态；**Output**（输出）包含非命令的 **System**、**Agent**、**Remote** 和 **Jobs** 日志；**Plot**（绘图）显示可用的结果图。 | 先看 Activity 和相关的 Output 来源，再按任务需要查看条目、任务结果面板、报告或 Plot。 |
| 右侧停靠区 | 默认打开 **Assistant**（助手），用于以对话方式准备或发起受支持的操作。 | 助手发起的工作仍在 Activity、相关的 Console 或 Output 记录及相应结果位置中核对。 |

## 区分三种状态

- **Entry selection**（条目选择）是 Entries 中被选中的一行或多行，用于条目级操作。
- **当前条目**是已载入中央视口、作为当前结构上下文的单个条目。单击某一行可以只改变条目选择；双击条目可将其激活。
- **Atom selection**（原子选择）是当前条目内被选中的原子，供 Style 和结构编辑等操作使用。

三者有时会同时指向同一结构，但不要据此把它们当成同一种状态。

## 恢复默认停靠布局

首次启动时，主侧栏默认显示 Entries；底部依次包含 Console、Sequence、
Activity、Output 和 Plot，其中 Console 处于打开状态；右侧打开 Assistant。

如果持久化布局难以辨认，请选择 **View > Reset Workbench Layout**（视图 >
重置工作台布局）。该命令恢复默认停靠区，但保留重置前的主侧栏视图，因此随后应再选择
**Entries**。

## 查找操作结果

任务结果不采用单一路径：任务可能创建新条目、原位修改结构、只写出文件，或生成报告。
操作后应检查 **Activity** 的状态、**Entries** 的变化，以及任务结果面板、报告、
**Plot** 或相关日志界面。**Console** 包含命令记录并提供 `sls>` 输入；**Output** 包含
非命令的 **System**、**Agent**、**Remote** 和 **Jobs** 日志，并不是 Console 的副本。

## 限制

界面状态和任务的 **Completed**（已完成）标记不验证方法选择、计算质量或科学结论。
